# OCR 识别优化指南

## 概述

本文档介绍主流程中针对 OCR 识别效果的优化措施。优化聚焦于**图像预处理**环节，通过改善输入图像质量来提升 OCR 识别准确率。

## 优化原理

OCR 识别质量主要受以下因素影响：
1. **对比度**：文字与背景的对比度越高，识别越准确
2. **清晰度**：图像噪声和模糊会降低识别率
3. **亮度**：过暗或过亮的图像难以识别
4. **文字边缘**：清晰的边缘有助于文字检测和识别

## 主流程优化点

在现有流程 `分割 → 透视校正 → OCR识别` 的基础上，新增了**图像预处理**环节：

```
原始图片 → YOLO分割 → 透视校正 → [图像预处理] → 方向矫正 → PaddleOCR → 结果输出
```

### 预处理时机

预处理在**透视校正之后、OCR识别之前**进行，原因如下：
- ✅ 透视校正后的图像已经是正面矩形，便于统一处理
- ✅ 预处理只作用于识别区域，避免影响分割和校正
- ✅ 可针对每个检测到的快递单独立优化

## 预处理功能详解

### 1. 对比度增强（CLAHE）

**功能**：提升局部对比度，使文字更加清晰

**适用场景**：
- 光照不均匀的图像
- 文字与背景对比度低
- 阴影或反光导致局部过暗/过亮

**参数**：
- `contrast_clip_limit`: 对比度限制阈值（推荐 2.0-2.5）
  - 值越大，对比度增强越明显
  - 过大可能导致噪声放大

**配置示例**：
```yaml
preprocessing:
  enhance_contrast: true
  contrast_clip_limit: 2.0
```

### 2. 去噪处理

**功能**：减少图像噪声，提升识别稳定性

**适用场景**：
- 低光环境拍摄的图像
- 压缩损伤的图像
- 存在颗粒噪声的扫描件

**参数**：
- `denoise_strength`: 去噪强度（推荐 5-10）
  - 值越大，去噪越强但可能模糊细节
  - 建议使用中等强度，平衡去噪和清晰度

**配置示例**：
```yaml
preprocessing:
  denoise: true
  denoise_strength: 7
```

### 3. 锐化处理

**功能**：增强边缘，使文字轮廓更清晰

**适用场景**：
- 模糊或失焦的图像
- 低分辨率图像
- 文字边缘不够清晰

**注意**：
- ⚠️ 锐化可能放大噪声，建议配合去噪使用
- ⚠️ 过度锐化会产生光晕效应
- 建议先测试，确认有效果再启用

**参数**：
- `sharpen_strength`: 锐化强度（推荐 0.5-1.0）

**配置示例**：
```yaml
preprocessing:
  sharpen: false  # 默认关闭，按需启用
  sharpen_strength: 0.5
```

### 4. 亮度调整

**功能**：将图像亮度调整到最佳识别范围

**适用场景**：
- 过暗的图像（欠曝）
- 过亮的图像（过曝）
- 需要统一亮度的批量图像

**参数**：
- `target_brightness`: 目标亮度值（推荐 120-140）
  - 128 为中等亮度
  - 根据实际测试调整

**配置示例**：
```yaml
preprocessing:
  adjust_brightness: false  # 按需启用
  target_brightness: 130
```

### 5. 二值化处理

**功能**：将图像转为纯黑白，极致对比度

**适用场景**：
- 简单的黑白文档
- 背景单一的快递单

**注意**：
- ⚠️ 会丢失颜色信息
- ⚠️ 可能损失部分浅色文字
- 建议谨慎使用，先尝试其他方法

**配置示例**：
```yaml
preprocessing:
  binarize: false  # 默认关闭
```

## 使用模式

### 模式 1：自动模式（推荐）

**特点**：系统自动评估图像质量，选择最优预处理方案

**配置**：
```yaml
preprocessing:
  enabled: true
  mode: "auto"
```

**适用场景**：
- 批量处理不同质量的图像
- 不确定最优参数时
- 快速上手使用

**自动策略**：
- 评估图像亮度：过暗或过亮则调整
- 评估对比度：低对比度则增强
- 始终应用轻度去噪（强度 5）
- 不启用锐化（避免过度处理）

### 模式 2：自定义模式

**特点**：完全自定义每个预处理步骤和参数

**配置**：
```yaml
preprocessing:
  enabled: true
  mode: "custom"
  enhance_contrast: true
  denoise: true
  sharpen: false
  adjust_brightness: false
  binarize: false
  contrast_clip_limit: 2.0
  denoise_strength: 7
  sharpen_strength: 0.5
  target_brightness: 130
```

**适用场景**：
- 已知图像质量特征
- 需要针对性优化
- 追求最佳识别效果

### 模式 3：关闭预处理

**配置**：
```yaml
preprocessing:
  enabled: false
  # 或
  mode: "off"
```

**适用场景**：
- 高质量原始图像
- 需要对比预处理效果
- 性能优先场景

## 推荐配置方案

### 方案 1：通用场景（默认配置）

适用于大多数快递单图像，平衡效果和性能。

```yaml
preprocessing:
  enabled: true
  mode: "auto"
```

### 方案 2：低质量图像

适用于模糊、噪点多、光照差的图像。

```yaml
preprocessing:
  enabled: true
  mode: "custom"
  enhance_contrast: true
  denoise: true
  sharpen: true
  adjust_brightness: true
  binarize: false
  contrast_clip_limit: 2.5
  denoise_strength: 10
  sharpen_strength: 0.8
  target_brightness: 130
```

### 方案 3：高质量图像

适用于清晰、光照均匀的图像，只做轻度优化。

```yaml
preprocessing:
  enabled: true
  mode: "custom"
  enhance_contrast: true
  denoise: true
  sharpen: false
  adjust_brightness: false
  binarize: false
  contrast_clip_limit: 1.8
  denoise_strength: 5
```

## 性能影响

各预处理步骤的性能开销（相对于原始处理时间）：

| 步骤 | 性能开销 | 建议 |
|------|---------|------|
| 对比度增强 | +5-10% | 推荐开启 |
| 去噪处理 | +15-25% | 推荐开启 |
| 锐化处理 | +3-5% | 按需开启 |
| 亮度调整 | +3-5% | 按需开启 |
| 二值化 | +5-8% | 谨慎开启 |

**总体影响**：
- 自动模式：约增加 20-30% 处理时间
- 完整自定义模式（全开）：约增加 30-50% 处理时间
- **收益**：OCR 识别准确率提升 10-30%（取决于原始图像质量）

## 效果评估

### 识别准确率提升场景

预处理对以下情况效果显著：

1. **光照不均匀** → 对比度增强 ✅
2. **图像模糊** → 去噪 + 锐化 ✅
3. **过暗/过亮** → 亮度调整 ✅
4. **背景复杂** → 对比度增强 ✅
5. **低分辨率** → 锐化 ✅

### 无明显改善场景

以下情况预处理效果有限：

1. **文字严重遮挡** → 需要模型改进 ❌
2. **极度模糊（失焦）** → 需要重新拍摄 ❌
3. **文字缺损** → 无法恢复 ❌
4. **极低分辨率（<100px）** → 需要高分辨率图像 ❌

## 调试与优化

### 1. 对比测试

建议测试步骤：
```bash
# 1. 关闭预处理，查看原始效果
# 修改 config/local.yaml:
preprocessing:
  enabled: false

python run_ocr.py test_image.jpg

# 2. 启用自动模式
preprocessing:
  enabled: true
  mode: "auto"

python run_ocr.py test_image.jpg

# 3. 对比识别结果和置信度
```

### 2. 查看中间结果

所有中间图像保存在输出目录：
```
output/图片名/
├── yolo_annotated.jpg      # YOLO 分割结果
├── 0_process.jpg           # 校正过程（含预处理前后对比）
├── 0_rectified.jpg         # 预处理后的图像
├── 0_ocr_boxes.jpg         # OCR 检测框
└── 0_result.json           # 识别结果（含置信度）
```

### 3. 参数微调建议

根据识别结果调整参数：

**问题：文字识别不全**
- → 降低 `det_db_thresh`: 0.2 → 0.15
- → 增大 `det_db_unclip_ratio`: 1.8 → 2.0
- → 启用对比度增强

**问题：识别错误率高**
- → 增强对比度：`contrast_clip_limit` 2.0 → 2.5
- → 增加去噪强度：`denoise_strength` 7 → 10
- → 启用锐化：`sharpen: true`

**问题：置信度低**
- → 启用自动模式
- → 调整亮度：`adjust_brightness: true`
- → 增强对比度

## 最佳实践

1. **从自动模式开始**
   - 先用 `mode: "auto"` 测试效果
   - 根据结果决定是否需要自定义

2. **逐步调整**
   - 一次只调整一个参数
   - 观察效果变化
   - 记录最佳参数

3. **批量测试**
   - 用代表性的图像样本测试
   - 评估整体识别率
   - 选择平衡方案

4. **保持简单**
   - 不要启用所有功能
   - 只开启必要的预处理
   - 避免过度优化

5. **监控性能**
   - 关注处理时间
   - 平衡准确率和速度
   - 根据场景调整

## 常见问题

**Q: 预处理会不会降低高质量图像的识别效果？**

A: 不会。自动模式会评估图像质量，只在需要时应用强处理。高质量图像只会应用轻度优化。

**Q: 所有预处理都开启效果最好吗？**

A: 不一定。过度处理可能引入伪影和噪声。建议从自动模式或推荐配置开始。

**Q: 预处理可以修复严重模糊的图像吗？**

A: 有限。预处理可以改善轻度模糊，但无法恢复严重失焦的图像。建议重新拍摄。

**Q: 预处理需要 GPU 吗？**

A: 不需要。所有预处理都在 CPU 上高效运行，性能开销可接受。

**Q: 如何评估预处理效果？**

A: 对比以下指标：
1. OCR 识别准确率
2. 文字检测框数量
3. 平均置信度
4. 处理时间
